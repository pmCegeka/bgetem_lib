Details zu den Stufen der Barrierefreiheitstests

1. Automatische Tests

Automatische Tests decken viele WCAG-Kriterien ab:

  • Alt-Attribute
  • Farbkontraste
  • HTML-Struktur
  • ARIA-Attribute
  • Tastaturbedienung

Tools: WAVE, Lighthouse

2. UX-Test (Manuelle Überprüfung)

  • Logische Navigation
  • Verständlichkeit von Inhalten
  • Fokus-Indikatoren sichtbar
  • Fehlerbehandlung hilfreich
  • Konsistenz der Bedienung

3. 200 % / 400 % Zoom

  • Lesbarkeit ohne horizontales Scrollen
  • Funktionalität bleibt erhalten
  • Test in verschiedenen Browsern

4. Kontrastmodus (High Contrast Mode)

  • Textkontrast: mind. 4.5:1
  • Nicht-Text-Kontrast: mind. 3:1
  • Keine reine Farbkommunikation

5. Screen Reader

Screen Reader wie NVDA, VoiceOver, JAWS oder EYE ABLE prüfen Lesbarkeit, Struktur und Fokusmanagement.

Ergänzungen für den Testprozess

  • Testumgebung: verschiedene Browser & Betriebssysteme
  • Dokumentation aller Tests und Korrekturen
  • Regelmäßige Wiederholungen
  • Schulung der Beteiligten
